# About these docs

The files in this directory are package documentation for the Note method: adopter-facing guides that explain, walk through, and restate what the corpus fixes.
A doc is plain Markdown with a level-1 title and no frontmatter envelope: docs are not artifacts, carry no `id`, `kind`, or `status`, and live outside the layer root, which holds authored intent only.
A doc cites corpus artifacts with the standard citation form, and the normative source is always the cited artifact, never the doc that restates it.
