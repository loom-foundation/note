The level places a Requirement on the refinement axis from the obligation as a stakeholder holds it to the obligation placed on the system being built.

It is orthogonal to Scope (where intent applies), to kind (need / requirement / specification), and to type (functional / non-functional / constraint).

The default vocabulary has two values; a profile may declare a richer ordered set where the context earns it (see Proportionality and tiered conformance).

The level is a label on a node of the derivation graph, not a structure of its own; the refinement itself is carried by the derivedFrom relation (see Derivation).
