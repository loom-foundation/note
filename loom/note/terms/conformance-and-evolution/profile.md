A named rigor bundle that fixes the depth of discipline a corpus, or a scope within it, declares it adopts: bare, standard, or strict.

A Profile carries no kind availability; the kinds legal in a corpus are determined solely by the Kind Packages the manifest enables.
Conformance is evaluated against the declared Profile, not against the maximal rule set.
Adoption is monotonic: raising a Profile may require adding sections, never invalidates an artifact valid under a lighter Profile, and never changes identity or breaks a relation.
A Profile is breadth of discipline; how far an artifact has matured is its status.

## Inclusion test

Is it a declared rigor depth that conformance is then judged against, without carrying any claim about which kinds exist?
If so it is a Profile.
If instead it is a selection of which kinds are legal in the corpus, that belongs to the pack mechanism, not to a Profile.

## Origin

Oxford: "the representation of something in outline", and by extension a defined set of characteristics or settings.
The usage follows the standards sense, where a profile is a named, conformant subset of a larger specification chosen for a particular use (as in ISO and W3C profiles); the profile fixes the rigor depth an implementation declares and is then measured against.
