A named depth of adopted discipline.

Tiers are additive: raising a tier never changes an artifact's identity, never breaks an existing relation, and never requires moving or restructuring files; it may require adding fields or enriching content.

## Inclusion test

Does raising it leave every existing identity and link intact?
If so it is a tier change; if raising it would break a link or change an identity, that is a migration, not a tier change.

## Origin

General English (Oxford: "one of a series of rows or levels placed one above another").

The term names a graded depth that stacks additively, each level resting on the ones below; the corpus keeps the metaphor strict, so moving up a tier only adds and never disturbs what a lower tier established.
