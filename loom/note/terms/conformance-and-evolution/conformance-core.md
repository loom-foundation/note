The small set of rules that determine whether a file is a valid artifact at all (well-formed frontmatter, required fields, unique identity, resolvable and type-compatible relations).

Everything beyond the core is opt-in.

## Inclusion test

Without this rule satisfied, is the file simply not a Note artifact?
If so it is core; if the file is still a valid artifact without it, the rule is opt-in.

## Origin

General English: "conformance" is compliance with a rule or standard, and a "core" is the central, essential part of a thing (Oxford).
The pairing names the essential, irreducible compliance every artifact owes, as distinct from the optional disciplines layered on top.

The usage tracks the standards sense of conformance, where a specification distinguishes mandatory requirements that define validity from recommended or optional ones (ISO/IEC directives, RFC 2119 "MUST" versus "SHOULD"/"MAY").
