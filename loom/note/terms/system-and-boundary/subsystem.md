A System that is a part of a larger system while remaining a coherent system in its own right.

In this project's instantiation, Note, embedded in a larger methodology-and-orchestration system, is a Subsystem of it: the same artifact as the standalone System, viewed from outside its boundary.

## Inclusion test

Is it both a coherent system on its own terms and a part of something larger in the current context?
If yes, it is a Subsystem in that context;
if instead it stands alone with nothing larger enclosing it, that is a System, and if it is an indivisible part with no independent coherence, that is a Component.

## Relations

- *is a* [System](system.md){id=note:term:0jbwfkm} (viewed from outside the boundary of the larger system that embeds it)

## Origin

General English (Oxford: "a system that is part of a larger system").

Standard in systems engineering through the INCOSE tradition, where a system-of-interest is relative to the observer's boundary, so the same arrangement is a system when viewed alone and a subsystem when viewed as an element of an enclosing system.
