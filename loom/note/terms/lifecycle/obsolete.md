No longer operative; kept for history, not cited for current decisions.

Where a named successor exists, the supersession is recorded with `supersedes` on the successor and `superseded_by` on the retired artifact.

## Inclusion test

Has it been retired, so it should be used only to reconstruct history?

If so, it is obsolete; if it is still the operative source, that is current.

## Relations

- *is a value of* [Status](status.md){id=note:term:xcen4v2}

## Origin

The Status value-vocabulary, naming the state of an artifact that has been retired and is kept only to reconstruct history.
Document control (ISO 9001:2015 §7.5) settles on `obsolete` as the term for a document no longer operative, with `superseded` the narrower replacement-by-a-named-successor case, preserved here in the pointer-field names.

A `deprecated` state is deliberately deferred until a corpus first needs to mark something deprecated rather than obsolete.
