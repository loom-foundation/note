A statement of what the system shall *do*: a behavior or capability.

## Inclusion test

Does it describe an action or capability the system performs, naming no mechanism?
If so it is Functional; if instead it states a quality of the whole rather than a discrete behavior, that is non-functional.

## Relations

- *is a value of* [Type](type.md){id=note:term:mwwzghp}
- *contrasts with* [Non-Functional](non-functional.md){id=note:term:a3gg1dj} (states a quality the system exhibits rather than a discrete behavior)

## Origin

The functional / non-functional split is standard in software requirements practice; a functional requirement specifies a behavior the system shall exhibit.

The usage follows INCOSE and ISO/IEC/IEEE 29148, where functional requirements describe what the system does, as distinct from how well it does it.
