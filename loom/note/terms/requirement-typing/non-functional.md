A statement of a quality the system shall *exhibit*: portability, durability, legibility, performance, security, and the other qualities.

## Inclusion test

Does it describe a quality of the whole rather than a discrete behavior, naming no mechanism?
If so it is Non-functional; if instead it describes an action or capability the system performs, that is functional.

## Relations

- *is a value of* [Type](type.md){id=note:term:mwwzghp}

## Origin

The functional / non-functional split is standard in software requirements practice; ISO/IEC 25010 names the quality attributes a system exhibits.

A non-functional requirement constrains how well the system does what it does rather than what it does, the quality axis to functional's behavior axis.
