Requirements a human and an AI read the same way.

## The case

A requirement is only as good as the words it is written in, and whoever reads it next builds to what those words say.
Write each one plainly, state the condition that decides whether it is met, and leave the design out, and a person and a model read it the same way.
Then either one can check it, and a yes-or-no answer is already there to be found rather than argued over.
