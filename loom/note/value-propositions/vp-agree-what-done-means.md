Agree what "done" means before anyone builds.

## The case

Write down what success looks like for each requirement before work starts, in plain words anyone can read and sign off on.
That agreement stands on its own, separate from the test that later proves it, so it is there before the build and still there after.
When a delivered feature is disputed, you point to what everyone agreed up front instead of arguing over a check written afterwards.
