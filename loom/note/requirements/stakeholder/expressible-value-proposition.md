An author shall be able to state the value a product or service offers a persona, as relief for a pain or a gain it creates, as intent that links to the need it answers and to the obligations that deliver it, so that the value is captured once and serves the requirements, the product copy, and the documentation alike.

## Source

See the Source of [Articulate value offered](../../needs/articulate-value-offered.md){id=note:need:9w2k4pt}.

## Rationale

A need states a persona's pains and gains, and a requirement states a solution-free obligation; neither states, in the customer's terms, the relief or gain the system offers in answer.

That value statement is distinct intent: it is customer-facing rather than technical, it is the thing marketing and documentation restate, and it is the link that makes a requirement's purpose legible.

The offered value is the Offering; the customer-facing pitch of it, when one is worth holding, is a Value Proposition.

This obligation requires only that such a value statement be expressible and linkable, and leaves whether it is a distinct kind, how it relates to needs and requirements, and how customer-facing surfaces derive from it to the system requirement and to specification.
